## Today (`/`)

**File:** `src/screens/today/TodayScreen.tsx`

The mobile dashboard (rendered at `/` when the viewport is below 768 px; on wider viewports `/` renders the desktop `OverviewScreen` instead). Shows habits due today, the week strip, and overall progress.

### Component tree

```
TodayScreen
  [scrollable area]
    header              -- date eyebrow, greeting, tagline (inline)
    summary card        -- Ring (today's percent) + current/best streak (inline)
    week strip          -- 7-day row, Mon to Sun, completion-aware fill (inline)
    HabitRow[]          -- today's scheduled habits; empty-state text if none
    + FAB               -- floating "add habit" button -> /create
```

The progress ring is the shared [`Ring`](/components/#ring) chart used directly; the header, summary card, and week strip are rendered inline in `TodayScreen` rather than as separate components.

### HabitRow

`src/screens/today/HabitRow.tsx`

- **Tap**: toggles today's completion (via `useToggleCompletion`)
- **Long-press (500 ms)**: opens the `HabitSheet` for this habit (via `useUIStore.openSheet`, using `useLongPress`)
- **Completion state**: accent-tinted background, strikethrough name, checkmark, animated sweep keyed off the habit's `hex`
- Shows: habit icon, name, schedule label, streak

`HabitRow` is also reused by the Habits screen (`src/screens/habits/HabitsScreen.tsx`).

## Habit Creator (`/create`)

**File:** `src/screens/creator/HabitCreatorScreen.tsx`

Form for adding a new habit.

- Name text input
- Icon picker: 25-emoji grid (`IconPicker`); selected icon highlighted
- Schedule selector: "Every day" or "Weekly (Mon)"
- On submit: creates a `Habit` with a fresh `crypto.randomUUID()` ID, `createdAt: todayStr()`, then navigates to `/`

## Calendar (`/calendar`)

**File:** `src/screens/calendar/CalendarScreen.tsx`

Year-at-a-glance heatmap of completion rates across all habits.

- The heatmap is rendered inline in `CalendarScreen` (12 month grids of day cells), not via a separate component. Day data comes from `useYearHeatmap()`.
- Each day cell is gradient-filled: a `linear-gradient` with `color-mix(in srgb, var(--accent) 40%, var(--surface-2))` rising to the day's completion percentage. Future days and 0% days show the empty surface color.
- Tapping a day opens a detail card with a `Ring` showing that day's percentage and the count of habits completed.

## Analytics (`/analytics`)

**File:** `src/screens/analytics/AnalyticsScreen.tsx`

| --------------------- | ----------------------------------- |

## Achievements (`/achievements`)

**File:** `src/screens/achievements/AchievementsScreen.tsx`

Five milestone badges. Each badge has a condition evaluated from live store data:

| Badge        | Condition                              |
| ------------ | -------------------------------------- |
| First Step   | At least 1 total completion            |
| Week Warrior | Current streak >= 7 days               |
| Iron Will    | Current streak >= 30 days              |
| Dedicated    | 10 or more completions on a single day |
| Centurion    | 100 or more total completions          |

Unlocked badges are shown in full color with a green checkmark. Locked badges are grayscale.

## Habit Detail (`/habit/:id`)

**File:** `src/screens/detail/HabitDetailScreen.tsx`

Full view for a single habit.

- Header: back button, habit icon, name, schedule label
- Stats row: current streak, total completion count
- Completion history: scrollable list, reverse-sorted by date, showing date and any note
- **Delete button**: calls `deleteHabit(id)` and navigates to `/`
- Shows a 404 message if the habit ID is not found in the store

This screen is also reachable from `CompletionLogTab` inside the `HabitSheet`.
