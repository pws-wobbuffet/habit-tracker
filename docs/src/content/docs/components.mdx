Shared components live in `src/components/` grouped into four subdirectories. Screen-specific subcomponents (like `HabitCard`) live alongside their screen in `src/screens/`.

## Layout

The app picks one nav at runtime based on a 768 px breakpoint: `SideNav` on desktop, `FloatingNav` on mobile. See [Architecture](/architecture/) for the responsive split.

### `SideNav`

**File:** `src/components/layout/SideNav.tsx`

The desktop (>= 768 px) left sidebar. Renders the profile, a column of nav items (Overview, Analytics, Calendar, Habits, Awards, Settings) using the icon set from `src/components/icons`, and a theme cycle button (auto / light / dark). The active item is highlighted using the `accent` color, derived from `useLocation()`.

### `FloatingNav`

**File:** `src/components/layout/FloatingNav.tsx`

The mobile (< 768 px) navigation: a fixed, blurred glass pill centered at the bottom of the viewport. Five tabs (Today, Calendar, Stats, Awards, Settings) each with an inline SVG icon, plus a sliding indicator whose left offset is computed from the active tab index. Sits above the safe-area bottom inset via `bottom: calc(var(--safe-bottom) + 14px)`.

## Sheets

The sheet pattern is: `HabitSheet` (logic + content) wraps `Sheet` (slide-up panel + backdrop). It is rendered as a portal on `document.body` from `App.tsx` when `activeHabitId` is set.

### `Sheet`

**File:** `src/components/sheets/Sheet.tsx`

**Props:**

```ts
{ open: boolean; onClose: () => void; children: ReactNode; maxHeight?: string }
```

A bottom-anchored slide-up panel:

- Translates in from the bottom (`translateY(101%)` to `translateY(0)`) on `open`, transitioning with `var(--ease-spring)`
- Semi-transparent, blurred backdrop; tapping it calls `onClose`
- Shows a handle bar at the top and pads its bottom with `var(--safe-bottom)`
- `maxHeight` defaults to `85dvh`

### `HabitSheet`

**File:** `src/components/sheets/HabitSheet.tsx`

**Props:**

```ts
{
  habitId: string
}
```

Wraps `Sheet` and renders tabs for the selected habit:

| Tab   | Component          | What it does                                       |
| ----- | ------------------ | -------------------------------------------------- |
| Note  | `TextNoteTab`      | Add / view text notes on completions               |
| Voice | `VoiceMemoTab`     | Record and play back audio memos                   |
| Log   | `CompletionLogTab` | Browse all past completions; link to detail screen |

The Voice tab is only shown when `navigator.mediaDevices.getUserMedia` is available.

**`TextNoteTab`** (`sheets/tabs/TextNoteTab.tsx`): textarea to save a text note. Shows the last 10 notes with their dates. Prevents duplicate same-day completions.

**`VoiceMemoTab`** (`sheets/tabs/VoiceMemoTab.tsx`): uses `MediaRecorder` to record audio. Clips are stored in local component state (not persisted to the backend yet). Shows a timer while recording and a list of recordings with their durations. Handles missing microphone permission gracefully.

**`CompletionLogTab`** (`sheets/tabs/CompletionLogTab.tsx`): reverse-sorted completion history. A "View full history" button navigates to `/habit/:id` and closes the sheet.

## UI

### `StreakChip`

**File:** `src/components/ui/StreakChip.tsx`

**Props:**

```ts
{ streak: number; size?: 'sm' | 'md' }
```

Renders `🔥 {streak}` in a small accent-colored pill. Returns `null` when `streak` is 0, so callers never need to conditionally render it.

### `FavoriteStar`

**File:** `src/components/ui/FavoriteStar.tsx`

**Props:**

```ts
{ active: boolean; onToggle: () => void }
```

A tap-animated star button (Framer Motion `whileTap: { scale: 0.8 }`). Renders a filled star when `active`, outline when not. Calls `event.stopPropagation()` so tapping it does not trigger the parent card's click handler.

### `ScheduleLabel`

**File:** `src/components/ui/ScheduleLabel.tsx`

**Props:**

```ts
{
  schedule: Schedule
}
```

Converts a `Schedule` value to a human-readable label:

- `'daily'` → "Every day"
- `'weekly'` → "Weekly"
- `{ days: [1, 3, 5] }` → "Mon, Wed, Fri"

### `Segmented`

**File:** `src/components/ui/Segmented.tsx`

**Props:**

```ts
{ options: { value: string; label: string }[]; value: string; onChange: (v: string) => void; small?: boolean }
```

A segmented control: a row of buttons over a sliding indicator whose width and `translateX` offset are computed from the option count and selected index. Used by `ScheduleEditor` and elsewhere for small two/three-way choices. Covered by a unit test in `src/components/ui/Segmented.test.tsx`.

### `ColorPicker`

**File:** `src/components/ui/ColorPicker.tsx`

**Props:**

```ts
{ value: string; onChange: (hex: string) => void }
```

A swatch grid over a fixed 8-color palette (Moss, Teal, Ocean, Iris, Amber, Rose, Clay, Sky). The selected swatch is scaled up and ringed. `value`/`onChange` are the habit's `hex` color.

### `EmojiPicker`

**File:** `src/components/ui/EmojiPicker.tsx`

**Props:**

```ts
{ value: string; onChange: (emoji: string) => void }
```

A grid of preset emoji for a habit's icon; the selected emoji is highlighted.

### `ScheduleEditor`

**File:** `src/components/ui/ScheduleEditor.tsx`

**Props:**

```ts
{ value: Schedule; onChange: (s: Schedule) => void }
```

Edits a habit's `Schedule`. A `Segmented` control switches between Daily, Weekdays, and Custom; in Custom mode it shows a Mon-Sun day toggle row that builds a `{ type: 'days', days: number[] }` value.

### `Toast`

**File:** `src/components/ui/Toast.tsx`

**Props:**

```ts
{ toast: Toast; onDismiss: (id: string) => void }
```

A single toast notification. Color-coded by type:

- `success` → green
- `info` → text color (neutral)
- `error` → red-600

Clickable to dismiss. Animated via Framer Motion layout.

### `ToastContainer`

**File:** `src/components/ui/ToastContainer.tsx`

No props. Reads `toasts` from `useUIStore` and renders each as a `Toast`. Fixed and centered near the bottom of the viewport (`bottom: calc(var(--safe-bottom) + 80px)`), clearing the `FloatingNav` and the safe-area bottom inset. Uses `AnimatePresence` with `mode="popLayout"` for smooth add/remove animations.

## Charts

All charts are presentational SVG / flexbox primitives in `src/components/charts/`. They take plain data arrays and default their color to `var(--accent)`. The calendar's year heatmap is **not** a chart component; it is rendered inline in `CalendarScreen` (see [Screens](/screens/)).

### `Ring`

**File:** `src/components/charts/Ring.tsx`

**Props:**

```ts
{ pct: number; size?: number; stroke?: number; color?: string; track?: string; children?: ReactNode }
```

An SVG progress ring. Draws a track circle and a foreground arc whose `strokeDashoffset` is computed from `pct` (clamped 0-100) and animated via a CSS transition. Centers its `children` (e.g. the percentage label). Used by `TodayScreen` for the today summary and by `CalendarScreen` for the selected-day detail.

### `Bars`

**File:** `src/components/charts/Bars.tsx`

**Props:**

```ts
{ data: Array<{ pct: number; done: number; due: number }>; height?: number; color?: string }
```

A flex row of vertical bars, one per data point, each scaled to its `pct`.

### `Area`

**File:** `src/components/charts/Area.tsx`

**Props:**

```ts
{ data: Array<{ pct: number }>; color?: string; width?: number; height?: number; fill?: boolean }
```

An SVG line/area sparkline over a series of percentages. Returns `null` for fewer than two points.

### `ConsBar`

**File:** `src/components/charts/ConsBar.tsx`

**Props:**

```ts
{
  pct: number
  color: string
}
```

A single horizontal consistency bar: a track with a fill whose width is `pct%`, animated via a CSS width transition.

### `DOW`

**File:** `src/components/charts/DOW.tsx`

**Props:**

```ts
{ data: Array<{ label: string; pct: number }>; color?: string }
```

A day-of-week bar chart. Bars are scaled relative to the highest `pct`, and the best day is emphasized.
