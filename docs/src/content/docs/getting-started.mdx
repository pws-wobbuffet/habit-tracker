## Installing as a PWA

<Tabs>
  <TabItem label="iOS (Safari)">
    <Steps>
      1. Open the app URL in **Safari** (Chrome on iOS cannot install PWAs).
      2. Tap the **Share** button (square with upward arrow).
      3. Scroll down and tap **Add to Home Screen**.
      4. Tap **Add**.
    </Steps>

    The app opens in standalone mode -- no browser chrome, full-screen, with system-level feel.

  </TabItem>
  <TabItem label="Android (Chrome)">
    <Steps>
      1. Open the app URL in **Chrome**.
      2. Tap the three-dot menu and choose **Add to Home screen**.
      3. Tap **Add**.
    </Steps>
  </TabItem>
  <TabItem label="Desktop (Chrome/Edge)">
    <Steps>
      1. Open the app URL.
      2. Click the install icon in the address bar (or use the browser menu).
      3. Confirm installation.
    </Steps>
  </TabItem>

## Running locally

### Prerequisites

| Tool                    | Version                | Install                                     |
| ----------------------- | ---------------------- | ------------------------------------------- |
| [bun](https://bun.sh)   | 1.3+                   | `curl -fsSL https://bun.sh/install \| bash` |
| [pnpm](https://pnpm.io) | 11+                    | `npm i -g pnpm` or via corepack             |
| A modern browser        | Chrome 90+, Safari 16+ | --                                          |

Node.js is **not** required; bun is the runtime for all scripts.

### Clone and run

```sh
git clone https://github.com/pws-wobbuffet/habit-tracker
cd habit-tracker
pnpm install
bun run dev
```

Open [http://localhost:5173](http://localhost:5173). The app auto-seeds sample habits on first launch in development.

### Available scripts

| Command             | What it does                                 |
| ------------------- | -------------------------------------------- |
| `bun run dev`       | Vite dev server with HMR at `localhost:5173` |
| `bun run build`     | Production build to `dist/`                  |
| `bun run preview`   | Serve `dist/` locally                        |
| `bun run lint`      | ESLint across all `.ts` / `.tsx` files       |
| `bun run format`    | Prettier check (use `--write` to auto-fix)   |
| `bun run typecheck` | TypeScript type-check without emitting       |
| `bun test`          | Run all tests with bun's test runner         |

## Common tasks

### Add a new screen

1. Create `src/screens/myscreen/MyScreen.tsx` with a default export.
2. Add a route in `src/App.tsx`:
   ```tsx
   <Route path="/myscreen" element={<MyScreen />} />
   ```
3. If it needs a nav entry, add it to `src/components/layout/FloatingNav.tsx` (mobile tabs) and/or `src/components/layout/SideNav.tsx` (desktop items).

### Add a new habit field

1. Update the `Habit` interface in `src/types/index.ts`.
2. Update the `StorageBackend` interface in `src/storage/types.ts` if the field needs its own query.
3. Update `src/storage/idb.ts` to persist the new field (IndexedDB stores full objects, so no schema migration needed for new optional fields).
4. Update `src/screens/creator/HabitCreatorScreen.tsx` to collect the value.

### Change the color palette

All palette tokens are CSS custom properties defined in `src/styles/globals.css` under `@theme`. Change there and the values propagate everywhere via Tailwind.

### Add a utility function

Place it in `src/lib/`. Keep it a pure function with no React dependencies. Export it directly (no default exports).

### Add a custom hook

Place it in `src/hooks/`. Name it `useXxx`. If it reads from a Zustand store, import the store hook directly inside -- do not pass the store as a parameter.

## Environment variables

Copy `.env.example` to `.env` before running:

```sh
cp .env.example .env
```

| Variable               | Default | Description                       |
| ---------------------- | ------- | --------------------------------- |
| `VITE_STORAGE_BACKEND` | `idb`   | `idb` or `remote`                 |
| `VITE_API_URL`         | (none)  | Required when backend is `remote` |

All `VITE_*` variables are inlined at build time by Vite. Changing them requires a rebuild.
